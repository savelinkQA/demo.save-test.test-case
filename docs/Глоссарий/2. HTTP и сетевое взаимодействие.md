# HTTP и сетевое взаимодействие

### POST-запрос
HTTP-метод для **отправки данных** на сервер (в отличие от GET для получения). В тест-кейсах проверяются запросы к эндпоинтам вида `/booking`, `/message` — отправка формы приводит к такому запросу.

### Статус-код (HTTP status code)
Числовой код ответа сервера на запрос:
- **200** — успех;
- **400** — ошибка запроса (невалидные данные, нарушение правил валидации);
- **409** — конфликт (например, номер уже забронирован на выбранные даты).

В ожидаемых результатах тестов явно указывается ожидаемый код (например, «POST запрос вернулся с 400 статус-кодом»).

### DevTools (инструменты разработчика)
Встроенные в браузер средства для отладки: просмотр DOM (вкладка **Elements**), мониторинг сетевых запросов (вкладка **Network**). В тест-кейсах используются для проверки факта отправки запроса и кода ответа.

### Вкладка Network
Раздел DevTools, в котором отображаются все HTTP-запросы и ответы страницы. Позволяет убедиться, что при нажатии «Забронировать»/«Отправить» уходит нужный запрос и приходит ожидаемый статус.
