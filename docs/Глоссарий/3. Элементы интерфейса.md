# Интерфейс и элементы UI

Краткий справочник по элементам интерфейса — термины, которые могут встречаться в тест-кейсах проекта



## Всплывающие элементы и окна

### Попап (popup)
Всплывающее окно поверх страницы — например, с подтверждением бронирования («Бронирование прошло успешно!»). В постусловиях часто требуется перезагрузка страницы, чтобы попап исчез.

### Модальное окно (modal)
Окно, которое **перекрывает** весь остальной интерфейс и блокирует взаимодействие с фоном до закрытия. Отличие от обычного попапа: пользователь должен явно закрыть окно (кнопка «Закрыть», крестик, клик вне области). В тестах проверяют закрытие по Esc, клику на overlay и кнопке отмены.

### Тултип (tooltip)
Короткая **всплывающая подсказка**, появляющаяся при наведении курсора на элемент (иконку, ссылку, кнопку). Исчезает при уходе курсора. В тестах проверяют появление при hover, корректный текст и отсутствие «залипания» тултипа.


## Элементы ввода и формы

### Поле ввода (input field)
Однострочное поле для ввода текста (логин, e-mail, телефон и т.д.). В тестах проверяют валидацию, маску ввода, ограничение длины и доступность (placeholder не заменяет label для скринридеров).

### Текстовое поле (textarea)
Многострочное поле для длинного текста (сообщение, комментарий). В тестах часто проверяют изменение размера (resize), скролл при переполнении и лимит символов.

### Плейсхолдер (placeholder)
**Подсказка внутри пустого поля** — серый текст, который исчезает при вводе (например, «Введите e-mail»). Не заменяет подпись поля (label): для доступности нужен и label, и placeholder при необходимости.

### Радиокнопка (radio button)
Элемент выбора **одного варианта из группы**: круглые кнопки с общим именем, активна только одна. В тестах проверяют переключение, невозможность снять выбор (если не предусмотрено) и корректную отправку выбранного значения.

### Чекбокс (checkbox)
Элемент для **независимого включения/выключения** опции. В отличие от радиокнопок, можно отметить несколько чекбоксов. В тестах проверяют состояние «отмечен/не отмечен», три состояния (checked / unchecked / indeterminate) при групповом выборе.

### Выпадающий список (dropdown / select)
Список вариантов, раскрывающийся по клику. Пользователь выбирает одно значение. В тестах проверяют открытие/закрытие списка, выбор пункта, отображение выбранного значения и поведение при поиске (если есть поиск внутри списка).

### Форма (form)
Набор полей ввода и кнопок для отправки данных (бронирование, отправка сообщения, регистрация). В тестах проверяют валидацию полей, отправку данных, сообщения об ошибках и сброс формы (постусловия).


## Навигация и структура

### Хлебные крошки (breadcrumb)
Цепочка ссылок вида «Главная → Каталог → Категория», показывающая путь по разделам сайта. В тестах проверяют корректность ссылок и текущую страницу (обычно последний элемент без ссылки).

### Вкладки (tabs)
Переключатели контента на одной странице: клик по вкладке показывает соответствующий блок, остальные скрыты. В тестах проверяют переключение, активное состояние и сохранение содержимого при смене вкладки.

### Гамбургер-меню (hamburger menu)
Иконка из **трёх горизонтальных линий**, открывающая скрытое меню (часто в мобильной версии). В тестах проверяют открытие/закрытие по клику, отображение пунктов и закрытие при клике вне меню.

### Аккордеон (accordion)
Блоки с заголовками: клик по заголовку **раскрывает** или **сворачивает** контент под ним. Часто используется в FAQ и фильтрах. В тестах проверяют открытие/закрытие, возможность одного или нескольких открытых блоков одновременно.


## Кнопки и действия

### CTA (Call to Action)
**Кнопка призыва к действию** — основная кнопка, на которую хотят направить пользователя («Купить», «Зарегистрироваться», «Отправить»). В тестах часто проверяют видимость, доступность, текст и переход/действие по клику.

### Тоггл (toggle / переключатель)
Элемент с двумя состояниями (вкл/выкл), обычно в виде «ползунка». В тестах проверяют смену состояния по клику, отображение текущего состояния и сохранение настройки при перезагрузке (если применимо).


## Прочее

### ЛКМ (левая кнопка мыши)
Стандартная кнопка мыши для клика и перетаскивания. В тест-кейсах используется при описании выбора периода в календаре («Зажать ЛКМ», «не отпуская ЛКМ, переместить курсор»).

### Скролл-бар (полоса прокрутки)
Элемент для прокрутки содержимого, когда оно не помещается в видимую область. В тестах проверяют появление/исчезновение при изменении размера, горизонтальный и вертикальный скролл.

### Оверлей (overlay)
Полупрозрачный слой поверх страницы (часто затемнение под модальным окном). Клик по оверлею часто закрывает модалку — это типичный сценарий для проверки.
