# Тестовые доступы

Сводка стендов и учётных записей для проверки бронирования номеров и отправки сообщений.
## Стенды

Окружения для проверок. Домены вымышленные, тематика — бронирование и гостиница.

| Стенд | URL | Назначение |
|-------|-----|------------|
| **Разработка** | https://dev.oteli-bron.ru | Ранние проверки фич, нестабильное окружение |
| **Тестирование** | https://test.oteli-bron.ru | Основной стенд для ручных и автотестов |
| **Предрелиз (stage)** | https://stage.oteli-bron.ru | Финальная проверка перед выкладкой в прод |

> **Важно:** на dev и test данные могут сбрасываться. Для стабильных сценариев используйте тестовый стенд; для проверки конфликтов бронирований — стенд, где есть предзаполненные номера и периоды.
## Учётные записи

Пароли — тестовые, только для стендов. В прод не использовать.

### Администратор

Полный доступ: управление номерами, бронированиями, настройками, просмотр сообщений.

| Поле | Значение |
|------|----------|
| Логин | `testadmin` |
| Пароль | `testadminpassword` |
| Роль | Администратор |

Используется в предусловиях кейсов, где требуется вход под администратором.
### Менеджер бронирований

Просмотр и редактирование бронирований, ответы на сообщения. Без доступа к настройкам системы и ролям.

| Поле | Значение |
|------|----------|
| Логин | `manager.bron` |
| Пароль | `Manager2025!` |
| Роль | Менеджер бронирований |

Подходит для проверки сценариев приёма заявок и работы с календарём.
### Гость (пользователь без входа)

Действия без авторизации: просмотр номеров, бронирование, отправка сообщения через форму.

| Поле | Значение |
|------|----------|
| Логин | — |
| Пароль | — |
| Роль | Не авторизован |

Используется для кейсов «Бронирование номера» и «Отправка сообщений», где вход не требуется.
### Тестовый пользователь (авторизованный гость)

Вход под обычным пользователем: свои бронирования, история, возможность отправлять сообщения от имени учётной записи.

| Поле | Значение |
|------|----------|
| Логин | `guest.test` |
| Пароль | `GuestTest99` |
| Роль | Пользователь |

Подходит для проверки личного кабинета и сценариев после авторизации.
## Краткая шпаргалка

| Роль | Логин | Когда использовать |
|------|--------|-------------------|
| Администратор | `testadmin` | Предусловия с админом, настройки, полный доступ |
| Менеджер | `manager.bron` | Работа с бронированиями и сообщениями |
| Пользователь | `guest.test` | Сценарии от имени авторизованного гостя |
| Гость | — | Бронирование и форма обратной связи без входа |
*Данные актуальны для стендов dev/test/stage. При смене паролей или URL обновите эту статью и тестовые предусловия.*
