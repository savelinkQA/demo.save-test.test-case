# Документация для проекта

Документация разрабатывается на языке разметки Markdown со свободной структурой. Файлы статей хранятся в общем репозитории проекта в папке `docs`.

В документации проекта рекомендуется указывать:
- Правила оформления тест-кейсов
- Полезные ссылки
- Тестовые аккаунты
- Глоссарий
- Подробные инструкции по настройке окружения
- Тестовые данные и прочее
